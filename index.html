How to make and publish a page on the Web

Here’s an example project you can use as reference while you go through the steps.

Ingredients

  • Text
  • Image
  • URL
  • HTML document
  • Colors
  • Typeface
  • Layout

Instructions

Add your content

  1. Find or write some text about a subject you’re interested in. Here’s an example. It will also be helpful to have at least one image and a URL to another website. Wikipedia may be a good place to find these.
  2. Download and open Sublime Text. (Or use your favorite code editor.)
  3. Create a new HTML document. and save it with the name “index.html”. That’s a special name that tells a web browser, “This is my home page”.
  4. Add your text in the space between the opening <body> tag and the closing </body> tag. You can also add the name of your image on a line by itself, and also the URL you found on a line by itself. This is your content.
  5. Mark up your content by following these examples. This will make the content understandable to software like search engines, screen readers, and web browsers.
  6. Open your document in your favorite web browser, to see how it’s looking.

Congrats! You just made a web page by hand.

Style your content (optional)

  1. Create an empty file with a “.css” extension. This will be your style sheet.
  2. Add a link to the style sheet in your HTML document.
  3. Open the style sheet and change the colors, margins and typeface to your liking.

Publish your page for the world to see!

GitHub Pages is a place where you publish your web page for free.

  1. Create a free account on GitHub.
  2. Create a new repository. User lowercase letters for the name of your repository, and use dashes instead of spaces. For example: “nuuk-greenland”
    Be sure to check the “Initialize this repository with a README” checkbox.
  3. Press the “Upload files” tab within your new repository.
  4. Upload your “index.html” file and any other files you created during this tutorial (for example, images and style.css).
  5. Scroll to the bottom of the page, write a commit message, and commit the new files.
  6. Press the “Settings” tab within your new repository.
  7. Scroll down to the GitHub Pages section. Then select the master branch source and press the Save button.
  8. Scroll down to the GitHub Pages section one more time, and press the link to your published project. The link should look a bit like this: http://username.github.io/repository

And you’re done!

You can make further changes to your files by uploading them again, or by visiting your repository and selecting a file to edit.

Ingredients

HTML document

<!doctype html>
<html lang="en" dir="ltr">
<head>
  <meta charset="utf-8" />
  <title></title>
</head>
<body>

</body>
</html>

Copy and paste this into a new file named “index.html”

Headline

<h1>The name of this page or section</h1>

Paragraph

<p>A section of text that represents a connected thought</p>

Link

<a href="The URL">The name of the link</a>

Image

<img src="The URL" alt="A caption for the image" />

Style

<link href="style.css" rel="stylesheet" />

Place this in the “head” element of your HTML document.

Color

body {
  background: black;
  color: white;
}

Typeface

body {
  font-family: Helvetica, sans-serif;
}

Layout

body {
  max-width: 40em;
  margin-left: auto;
  margin-right: auto;
}
